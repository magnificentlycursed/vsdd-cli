# CLI Supplement

Per-domain extensions for projects that ship a command-line interface (cargo / git / rustup / kubectl / docker ecosystem).

## UX extensions

- **Subcommand convention.** Top-level binary dispatches to subcommands (`vsdd init`, `vsdd verify check`, `vsdd observe cycle`). Matches cargo / git / rustup / docker ecosystem.
- **`--help` discipline.** Every command + subcommand has `--help` output naming the command's purpose, args, flags, examples. Auto-generated by argument-parser (clap / argparse / commander / cobra).
- **Flag conventions.** Long flags (`--verbose`) for clarity; short flags (`-v`) for frequent use; `--format json|yaml|sarif|compact` for output-format selection; `--files <path>...` for file-list args.
- **Exit code discipline.** 0 for success; non-zero for failure with per-code meaning. Per-error-code documentation (`vsdd verify explain <code>`); Rust `--explain` pattern.
- **Color-coded output.** TTY detection; ANSI escapes when TTY; plain output when piped. `--no-color` flag + `NO_COLOR` env var respected.
- **OSC-8 hyperlinks.** Terminal-emulator-capable hyperlinks for cross-references in CLI output (`https://gnu.org/software/screen/manual/html_node/Hyperlinks.html`).

## Software Engineer extensions

- **Argument parser choice.** Rust: `clap`. Python: `argparse` / `typer` / `click`. Go: `cobra`. Pick one; consistent across the project.
- **Subcommand dispatch.** Subcommand-tree at root level; each subcommand a module / package. Function-per-subcommand for testability.
- **Configuration file precedence.** Per-project config + per-user config + CLI args, where CLI args override per-project, which overrides per-user. Per-source loading is explicit.
- **stdin / stdout / stderr discipline.** Data on stdout; logs + errors on stderr; never mixed. Pipe-friendly tools.

## Quality Engineer extensions

- **Per-subcommand fixture.** `manual-tests/<binary>.md` documents per-subcommand acceptance criteria.
- **Snapshot testing.** Tools like `insta` (Rust) / `snapshot-testing-library` (JS) / `syrupy` (Python) for CLI-output regression testing.
- **End-to-end testing.** Invoke the binary as subprocess; assert stdout + stderr + exit code; per-subcommand smoke tests.
- **Pipe-compatibility testing.** Tests verify pipe-friendly stdout (machine-readable when piped) + TTY-friendly when run interactively.

## Accessibility extensions

- **`--no-color` discipline.** Every color-coded output respects `--no-color` flag + `NO_COLOR` env var.
- **Screen-reader-friendly output.** Long-running commands output progress to stderr (screen-reader-announced) not via terminal-cursor-tricks.
- **Keyboard-only.** Interactive prompts have keyboard-only navigation (arrow keys, Tab, Enter, Escape); never require mouse.
- **High-contrast.** Color choices tested with high-contrast terminal themes; not relying on color-alone for severity differentiation.
